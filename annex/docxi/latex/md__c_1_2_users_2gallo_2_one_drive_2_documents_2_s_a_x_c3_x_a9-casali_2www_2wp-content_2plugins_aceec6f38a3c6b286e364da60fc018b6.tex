\doxylink{class_p_h_p___code_sniffer}{PHP\+\_\+\+Code\+Sniffer} is a set of two PHP scripts; the main {\ttfamily phpcs} script that tokenizes PHP, Java\+Script and CSS files to detect violations of a defined coding standard, and a second {\ttfamily phpcbf} script to automatically correct coding standard violations. \doxylink{class_p_h_p___code_sniffer}{PHP\+\_\+\+Code\+Sniffer} is an essential development tool that ensures your code remains clean and consistent.

The easiest way to get started with \doxylink{class_p_h_p___code_sniffer}{PHP\+\_\+\+Code\+Sniffer} is to download the \href{http://php.net/manual/en/intro.phar.php}{\texttt{ Phar}} files for each of the commands\+: \begin{DoxyVerb}curl -OL https://squizlabs.github.io/PHP_CodeSniffer/phpcs.phar
php phpcs.phar -h

curl -OL https://squizlabs.github.io/PHP_CodeSniffer/phpcbf.phar
php phpcbf.phar -h
\end{DoxyVerb}


If you use PEAR, you can install \doxylink{class_p_h_p___code_sniffer}{PHP\+\_\+\+Code\+Sniffer} using the PEAR installer. This will make the {\ttfamily phpcs} and {\ttfamily phpcbf} commands immediately available for use. To install \doxylink{class_p_h_p___code_sniffer}{PHP\+\_\+\+Code\+Sniffer} using the PEAR installer, first ensure you have \href{http://pear.php.net/manual/en/installation.getting.php}{\texttt{ installed PEAR}} and then run the following command\+: \begin{DoxyVerb}pear install PHP_CodeSniffer
\end{DoxyVerb}


If you prefer using \href{http://getcomposer.org/}{\texttt{ Composer}} you can easily install \doxylink{class_p_h_p___code_sniffer}{PHP\+\_\+\+Code\+Sniffer} system-\/wide with the following command\+: \begin{DoxyVerb}composer global require "squizlabs/php_codesniffer=*"
\end{DoxyVerb}


Make sure you have the composer bin dir in your PATH. The default value is {\ttfamily \texorpdfstring{$\sim$}{\string~}/.composer/vendor/bin/}, but you can check the value that you need to use by running {\ttfamily composer global config bin-\/dir -\/-\/absolute}.
