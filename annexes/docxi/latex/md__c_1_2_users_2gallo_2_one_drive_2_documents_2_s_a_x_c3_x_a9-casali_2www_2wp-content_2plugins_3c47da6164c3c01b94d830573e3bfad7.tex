Once you have included the Php\+Spreadsheet files in your script, but before instantiating a {\ttfamily Spreadsheet} object or loading a workbook file, there are a number of configuration options that can be set which will affect the subsequent behaviour of the script.\hypertarget{md__c_1_2_users_2gallo_2_one_drive_2_documents_2_s_a_x_c3_x_a9-casali_2www_2wp-content_2plugins_3c47da6164c3c01b94d830573e3bfad7_autotoc_md724}{}\doxysection{\texorpdfstring{Cell collection caching}{Cell collection caching}}\label{md__c_1_2_users_2gallo_2_one_drive_2_documents_2_s_a_x_c3_x_a9-casali_2www_2wp-content_2plugins_3c47da6164c3c01b94d830573e3bfad7_autotoc_md724}
By default, Php\+Spreadsheet holds all cell objects in memory, but you can specify alternatives to reduce memory consumption at the cost of speed. Some localisation elements have been included in Php\+Spreadsheet. You can set a locale by changing the settings. To set the locale to Brazilian Portuguese you would use\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\ php}
\DoxyCodeLine{\$locale\ =\ 'pt\_br';}
\DoxyCodeLine{\$validLocale\ =\ \(\backslash\)PhpOffice\(\backslash\)PhpSpreadsheet\(\backslash\)Settings::setLocale(\$locale);}
\DoxyCodeLine{if\ (!\$validLocale)\ \{}
\DoxyCodeLine{\ \ \ \ echo\ 'Unable\ to\ set\ locale\ to\ '\ .\ \$locale\ .\ "{}\ -\/\ reverting\ to\ en\_us"{}\ .\ PHP\_EOL;}
\DoxyCodeLine{\}}

\end{DoxyCode}



\begin{DoxyItemize}
\item If Brazilian Portuguese language files aren\textquotesingle{}t available, then Portuguese will be enabled instead
\item If Portuguese language files aren\textquotesingle{}t available, then the {\ttfamily set\+Locale()} method will return an error, and American English (en\+\_\+us) settings will be used throughout.
\end{DoxyItemize}
