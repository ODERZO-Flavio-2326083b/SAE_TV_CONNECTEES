As you already know from the \href{./architecture.md\#readers-and-writers}{\texttt{ architecture}}, reading and writing to a persisted storage is not possible using the base Php\+Spreadsheet classes. For this purpose, Php\+Spreadsheet provides readers and writers, which are implementations of {\ttfamily \textbackslash{}Php\+Office\textbackslash{}Php\+Spreadsheet\textbackslash{}Reader\textbackslash{}IReader} and {\ttfamily \textbackslash{}Php\+Office\textbackslash{}Php\+Spreadsheet\textbackslash{}Writer\textbackslash{}IWriter}.\hypertarget{md__c_1_2_users_2gallo_2_one_drive_2_documents_2_s_a_x_c3_x_a9-casali_2www_2wp-content_2plugins_677dc02e54ad837d69ba537d1c2b49b7_autotoc_md585}{}\doxysection{\texorpdfstring{\textbackslash{}\+Php\+Office\textbackslash{}\+Php\+Spreadsheet\textbackslash{}\+IOFactory}{\textbackslash{}\+Php\+Office\textbackslash{}\+Php\+Spreadsheet\textbackslash{}\+IOFactory}}\label{md__c_1_2_users_2gallo_2_one_drive_2_documents_2_s_a_x_c3_x_a9-casali_2www_2wp-content_2plugins_677dc02e54ad837d69ba537d1c2b49b7_autotoc_md585}
The Php\+Spreadsheet API offers multiple methods to create a {\ttfamily \textbackslash{}Php\+Office\textbackslash{}Php\+Spreadsheet\textbackslash{}Reader\textbackslash{}IReader} or {\ttfamily \textbackslash{}Php\+Office\textbackslash{}Php\+Spreadsheet\textbackslash{}Writer\textbackslash{}IWriter} instance\+:

Direct creation via {\ttfamily \textbackslash{}Php\+Office\textbackslash{}Php\+Spreadsheet\textbackslash{}IOFactory}. All examples underneath demonstrate the direct creation method. Note that you can also use the {\ttfamily \textbackslash{}Php\+Office\textbackslash{}Php\+Spreadsheet\textbackslash{}IOFactory} class to do this.\hypertarget{md__c_1_2_users_2gallo_2_one_drive_2_documents_2_s_a_x_c3_x_a9-casali_2www_2wp-content_2plugins_677dc02e54ad837d69ba537d1c2b49b7_autotoc_md586}{}\doxysubsection{\texorpdfstring{Creating {\ttfamily \textbackslash{}Php\+Office\textbackslash{}Php\+Spreadsheet\textbackslash{}Reader\textbackslash{}IReader} using {\ttfamily \textbackslash{}Php\+Office\textbackslash{}Php\+Spreadsheet\textbackslash{}IOFactory}}{Creating {\ttfamily \textbackslash{}Php\+Office\textbackslash{}Php\+Spreadsheet\textbackslash{}Reader\textbackslash{}IReader} using {\ttfamily \textbackslash{}Php\+Office\textbackslash{}Php\+Spreadsheet\textbackslash{}IOFactory}}}\label{md__c_1_2_users_2gallo_2_one_drive_2_documents_2_s_a_x_c3_x_a9-casali_2www_2wp-content_2plugins_677dc02e54ad837d69ba537d1c2b49b7_autotoc_md586}
There are 2 methods for reading in a file into Php\+Spreadsheet\+: using automatic file type resolving or explicitly.

Automatic file type resolving checks the different {\ttfamily \textbackslash{}Php\+Office\textbackslash{}Php\+Spreadsheet\textbackslash{}Reader\textbackslash{}IReader} distributed with Php\+Spreadsheet. If one of them can load the specified file name, the file is loaded using that {\ttfamily \textbackslash{}Php\+Office\textbackslash{}Php\+Spreadsheet\textbackslash{}Reader\textbackslash{}IReader}. Explicit mode requires you to specify which {\ttfamily \textbackslash{}Php\+Office\textbackslash{}Php\+Spreadsheet\textbackslash{}Reader\textbackslash{}IReader} should be used.

You can create a {\ttfamily \textbackslash{}Php\+Office\textbackslash{}Php\+Spreadsheet\textbackslash{}Reader\textbackslash{}IReader} instance using {\ttfamily \textbackslash{}Php\+Office\textbackslash{}Php\+Spreadsheet\textbackslash{}IOFactory} in automatic file type resolving mode using the following code sample\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\ php}
\DoxyCodeLine{\$spreadsheet\ =\ \(\backslash\)PhpOffice\(\backslash\)PhpSpreadsheet\(\backslash\)IOFactory::load("{}05featuredemo.xlsx"{});}

\end{DoxyCode}


A typical use of this feature is when you need to read files uploaded by your users, and you don’t know whether they are uploading xls or xlsx files.

If you need to set some properties on the reader, (e.\+g. to only read data, see more about this later), then you may instead want to use this variant\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\ php}
\DoxyCodeLine{\$reader\ =\ \(\backslash\)PhpOffice\(\backslash\)PhpSpreadsheet\(\backslash\)IOFactory::createReaderForFile("{}05featuredemo.xlsx"{});}
\DoxyCodeLine{\$reader-\/>setReadDataOnly(true);}
\DoxyCodeLine{\$reader-\/>load("{}05featuredemo.xlsx"{});}

\end{DoxyCode}


You can create a {\ttfamily \textbackslash{}Php\+Office\textbackslash{}Php\+Spreadsheet\textbackslash{}Reader\textbackslash{}IReader} instance using {\ttfamily \textbackslash{}Php\+Office\textbackslash{}Php\+Spreadsheet\textbackslash{}IOFactory} in explicit mode using the following code sample\+:


\begin{DoxyCode}{0}

Xls file format is the old Excel file format, implemented in Php\+Spreadsheet to provide a uniform manner to create both .xlsx and .xls files. It is basically a modified version of \href{https://pear.php.net/package/Spreadsheet_Excel_Writer}{\texttt{ PEAR Spreadsheet\+\_\+\+Excel\+\_\+\+Writer}}, although it has been extended and has fewer limitations and more features than the old PEAR library. This can read all BIFF versions that use OLE2\+: BIFF5 (introduced with office 95) through BIFF8, but cannot read earlier versions.

Xls file format will not be developed any further, it just provides an additional file format for Php\+Spreadsheet.
