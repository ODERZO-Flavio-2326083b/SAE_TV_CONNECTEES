Parses portable object (PO) format.

From \href{http://www.gnu.org/software/gettext/manual/gettext.html\#PO-Files}{\texttt{ http\+://www.\+gnu.\+org/software/gettext/manual/gettext.\+html\#\+PO-\/\+Files}} we should be able to parse files having\+:

white-\/space \hypertarget{class_symfony_1_1_component_1_1_translation_1_1_loader_1_1_po_file_loader_autotoc_md774}{}\doxysubsection{\texorpdfstring{translator-\/comments}{translator-\/comments}}\label{class_symfony_1_1_component_1_1_translation_1_1_loader_1_1_po_file_loader_autotoc_md774}
\#. extracted-\/comments \#\+: reference... \#, flag... \#\texorpdfstring{$\vert$}{|} msgid previous-\/untranslated-\/string msgid untranslated-\/string msgstr translated-\/string

extra or different lines are\+:

\#\texorpdfstring{$\vert$}{|} msgctxt previous-\/context \#\texorpdfstring{$\vert$}{|} msgid previous-\/untranslated-\/string msgctxt context

\#\texorpdfstring{$\vert$}{|} msgid previous-\/untranslated-\/string-\/singular \#\texorpdfstring{$\vert$}{|} msgid\+\_\+plural previous-\/untranslated-\/string-\/plural msgid untranslated-\/string-\/singular msgid\+\_\+plural untranslated-\/string-\/plural msgstr\mbox{[}0\mbox{]} translated-\/string-\/case-\/0 ... msgstr\mbox{[}N\mbox{]} translated-\/string-\/case-\/n

The definition states\+:
\begin{DoxyItemize}
\item white-\/space and comments are optional.
\item msgid "{}"{} that an empty singleline defines a header.
\end{DoxyItemize}

This parser sacrifices some features of the reference implementation the differences to that implementation are as follows.
\begin{DoxyItemize}
\item No support for comments spanning multiple lines.
\item Translator and extracted comments are treated as being the same type.
\item Message IDs are allowed to have other encodings as just US-\/\+ASCII.
\end{DoxyItemize}

Items with an empty id are ignored.
