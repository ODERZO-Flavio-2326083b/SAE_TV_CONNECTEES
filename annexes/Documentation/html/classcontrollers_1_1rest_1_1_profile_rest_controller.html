◆ getItem()

controllers\rest\ProfileRestController::getItem ( $request)

Récupère les informations de l'utilisateur courant via l'API REST.

Cette méthode récupère les informations du profil de l'utilisateur actuellement connecté en interrogeant la base de données. Elle renvoie les données de l'utilisateur, telles que l'ID, le login, l'email, le rôle et les codes associés. Si l'utilisateur n'est pas trouvé, une réponse HTTP 404 avec un message d'erreur est retournée.

Paramètres
WP_REST_Request$requestRequête envoyée à l'API REST.
Renvoie
WP_REST_Response Réponse REST contenant les données utilisateur ou un message d'erreur.

controllers\rest\ProfileRestController::getItemPermissionsCheck ( $request)

Vérifie les permissions de l'utilisateur pour récupérer ses informations via l'API REST.

Cette méthode s'assure que l'utilisateur est actuellement connecté avant de permettre l'accès à ses données. Si l'utilisateur est connecté, l'accès est accordé, sinon, la requête est refusée.

Paramètres
WP_REST_Request$requestRequête envoyée à l'API REST.
Renvoie
bool Renvoie true si l'utilisateur est connecté, false sinon.

Enregistre les routes pour l'API REST "Profile".

Cette méthode utilise 'register_rest_route()' pour définir les routes disponibles dans l'API REST sous le namespace spécifié. Elle enregistre une route accessible en méthode "GET" pour récupérer des informations de profil d'utilisateur, avec une vérification des permissions et un schéma public.
