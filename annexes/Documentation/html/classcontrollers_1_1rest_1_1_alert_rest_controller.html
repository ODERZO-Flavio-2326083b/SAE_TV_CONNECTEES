◆ createItem()

controllers\rest\AlertRestController::createItem ( $request)

Crée une nouvelle alerte.

Cette méthode permet à un utilisateur d'ajouter une nouvelle alerte au système. Elle récupère les données fournies dans la requête, définit les propriétés de l'alerte, y compris l'auteur, le contenu, la date de création et la date d'expiration. Les codes ADE associés à l'alerte sont également définis. En cas d'erreur lors de la création de l'alerte, un message d'erreur approprié est renvoyé. Si l'insertion est réussie, l'ID de l'alerte créée est retourné.

Paramètres
WP_REST_Request$requestLes paramètres de la requête REST contenant les détails de l'alerte.
Renvoie
WP_REST_Response Une réponse de l'API REST contenant l'ID de l'alerte créée ou un message d'erreur.

◆ registerRoutes()

controllers\rest\AlertRestController::registerRoutes ( )

Enregistre les routes REST pour la gestion des alertes.

Cette méthode définit les différentes routes pour les opérations CRUD (Créer, Lire, Mettre à jour, Supprimer) sur les alertes dans l'API REST. Elle enregistre les routes pour obtenir la liste des alertes, créer une nouvelle alerte, récupérer une alerte spécifique par son identifiant, mettre à jour une alerte existante et supprimer une alerte. Pour chaque route, les méthodes, les callbacks, les contrôles de permission et les arguments requis sont spécifiés.

Renvoie
void
Version
1.0
Date

Met à jour une alerte existante avec les nouvelles données fournies.

Cette méthode récupère les détails d'une alerte spécifique à l'aide de son identifiant, puis met à jour son contenu, sa date d'expiration et ses codes ADE si de nouvelles informations sont fournies dans la requête.
